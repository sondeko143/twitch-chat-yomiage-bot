# chat 翻訳での Twitch emote パススルー — spec

- 日付: 2026-06-24 ／ 状態: 実装済み
- 関連 ADR: [0001 emote 除外を tcyb 側で行う](../../adr/0001-emote-exclusion-on-tcyb-side.md)

## 問題

`tcyb` の `ReadChat` は視聴者チャットを外部翻訳コマンドへ渡し、結果を `@reply-parent-msg-id` 付きで返信する。本文に Twitch emote（例 `DinoDance`）が含まれると翻訳器がその単語まで訳そうとし、結果が不自然になる。

## ゴール

翻訳クエリから Twitch ネイティブ emote を除外し、翻訳結果に emote を連結して返信する。

## 非ゴール

- TTS（読み上げ）は従来どおり元メッセージ全文を読み上げる。
- 外部翻訳コマンドのインターフェース（プレーンテキスト 1 引数）は変更しない。
- サードパーティ emote（BTTV / FFZ / 7TV）。IRC の `emotes` タグに載らないため対象外。
- 対象は IRC 経由の `ReadChat` 翻訳のみ。

## 受入基準

- [ ] Twitch ネイティブ emote（サブスク/グローバル/bits）を含むチャットで、翻訳クエリから emote が除外され、返信に emote が連結される。
- [ ] 同一 emote が複数回出現しても返信では 1 つにまとめられ、出現順が保たれる。
- [ ] emote のみのメッセージは翻訳コマンドを呼ばず、emote を空白区切りで連結して返信する。
- [ ] 本文＋emote のメッセージで、翻訳結果が非空なら「翻訳文 emote…」の順で返信する。翻訳結果が空なら従来どおり返信しない。
- [ ] 日本語＋emote（コードポイント境界）でも本文が壊れずに分割される。
- [ ] emote を含まない（`emotes=`）メッセージの返信は従来と完全に同一（後方互換）。
